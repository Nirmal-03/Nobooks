Multi Threading

1. Executing more than one thread simultaneously is called multithreading.

2.In multithreading each and every thread is a separate independent process of same application.

3.However, we use multithreading than multiprocessing because threads use a shared memory area. They don't allocate separate memory area so saves memory, and context-switching between the threads takes less time than process.

4.Java Multithreading is mostly used in games, animation, etc.


Inter-Thread Communication

Two threads can communicate which each other by using three methods

• Wait()
• Notify()
• NotifyAll()


1. The thread which is expecting updation is responsible to call wait()method, then immediately the thread will enter into waiting state.

2. The thread which is responsible to perform updation is also responsible to call Notify() method after performing updation.

3. Waiting thread will get that notification and continue its execution with those updated items.

4. Wait(),Notify(),NotifyAll() methods present in object class not in thread class.

5. To call wait(),Notify(),NotifyAll() methods on any object,Thread should have Lock of that object,i.e Thread should be inside Synchronized Block.

6. Illegal Monitor State Exception will be raised if wait(),Notify(),NotifyAll() methods called outside Synchronized Area.

7. If a thread calls wait () method on any object it immediately releases the Lock of that particular object and enter into the waiting state.

If a thread calls Notify() method on any object it releases the Lock of that object,but may not immediately

Polymorphism

One thing into many forms.
Polymorphism, in the context of object-oriented programming, refers to the ability of objects to take on different forms or behaviors. It allows different objects to be treated as if they belong to a common type, enabling code flexibility and reusability.

There are two types of polymorphism


1.Compile time polymorphism / Static binding:
We can achieve compile time polymorphism using Method overloading

2.Runtime polymorphism /Dynamic binding:
 We can achieve Run time polymorphism using Method overriding


Polymorphism is a Greek word


Poly - means – Many
Morphism – means - forms


Overloading

Three types of overloading in JAVA

  • Method overloading .
  • Counstructor overloading .
  • Operator overloading .

Method Overloading

If a class contains more than one method with the same name but different no of arguments or same name ,same no of arguments but different data type is called overloaded methods.

Constructor Overloading

If a class contains more than one constructor with different no of arguments or same no of arguments but different data type is called overloaded constructor.

Operator Overloading

If a class contains more than one constructor with different no of arguments or same no of arguments but different data type is called overloaded constructor.

  • One operator with more than one behavior is called operator overloading.
  • JAVA is not supporting operator overloading.
  • But implicitly there is a operator in java called + operator will take care by JVM to perform operator overloading JAVA is not supporting operator overloading.

Overloadinng Vs Overriding

  • To achieve overloading One JAVA class is enough….
  • To achieve overriding two classes is required with Inheritance.

Overriding in Java

  • In any object-oriented programming language, Overriding is a feature that allows a subclass or child class to provide a specific implementation of a method that is already provided by one of its super-classes or parent classes. 

  • When a method in a subclass has the same name, same parameters or signature, and same return type as a method in its super-class, then the method in the subclass is said to override the method in the super-class.
